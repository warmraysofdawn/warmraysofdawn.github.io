Программное решение для систематизации, сохранения и публикации творческих работ

pywebview

Идея создания

Множество творческих людей создают материалы, но часто сталкиваются с проблемой их упорядочивания и представления широкой аудитории.

🎯 Проблема

Существующие платформы требуют навыков разработки, навязывают стиль, не дают полного контроля или смешивают личное с профессиональным.

💡 Решение

Объединить функции каталогизатора, архиватора и генератора сайтов в одном простом приложении.

Кому это нужно?

🎨 Художники

Хранение изображений, описаний техник, создание галерей.

✍️ Писатели и поэты

Структурирование рассказов, стихов, романов с обложками.

🎵 Музыканты

Демо-записи, ноты, описания проектов.

💻 Программисты-любители

Код, скриншоты, документация к проектам.

🔬 Научные сотрудники

Публикации, учебные материалы, презентации.

📚 Школьники и студенты

Портфолио достижений для конкурсов и олимпиад.

Готовое приложение

Изначально планировалась библиотека для разработчиков, но анализ показал: большинство пользователей — не программисты.

🚀 Запуск двойным щелчком

Не требует установки интерпретаторов, управления зависимостями, знания кода.

🖥️ Графический интерфейс

Интуитивные формы заполнения, встроенный веб-сервер и база данных скрыты от пользователя.

Каталогизация работ

При добавлении работы указываются:

Название, описание, дата
Сфера → Жанр → Тема (единообразная классификация)
Произвольные теги
Основной контент, обложка, README (Markdown)
Автоматическое переименование файлов + SHA-256 хеш

Визуализация связей

Поле «Связанные ID» создаёт граф зависимостей между работами (серии рассказов, модули проекта).

Пример: связи между работами разных сфер (литература, программирование, искусство)

Настройка портфолио

Фото, имя, цитата, биография
Цветовая акцентная гамма
Выбор шаблона из папки static/templates
SEO-метатеги для генерируемых страниц

Галереи

Создание галерей с множеством изображений, названием, описанием, типом отображения (сетка/список). Изображения хешируются.

Надёжность данных

💾 Автоматические бэкапы

ZIP-архивы всех данных и файлов при каждом изменении, ротация (10 копий), ручное создание с комментарием.

🔍 Проверка целостности

Сравнение SHA-256 хешей, выявление отсутствующих или повреждённых файлов, восстановление из бэкапа.

Публикация и версионирование

📦 Экспорт сайта

Генерация полноценного статического сайта (HTML, «красивые» URL, .htaccess/_redirects) в ZIP-архиве.

🐙 Git-интеграция

Опционально: подключение GitHub/GitLab, история коммитов, push/pull, восстановление файлов из прошлых версий.
